# GitHub Pages 배포 가이드

## 이 방식이 맞는 경우
- 아이폰에 설치해서 어디서든 쓰고 싶음
- 데이터는 각 기기에 따로 저장되어도 괜찮음
- 실시간 동기화는 지금 당장 필요 없음

## 배포 전 확인
- 이 폴더는 아직 Git 저장소가 아닙니다.
- GitHub Pages는 정적 파일만 배포합니다.
- 따라서 이 앱은 [`index.html`](/Users/sonjisun/자동화/모임총무/index.html), [`app.js`](/Users/sonjisun/자동화/모임총무/app.js), [`styles.css`](/Users/sonjisun/자동화/모임총무/styles.css), [`service-worker.js`](/Users/sonjisun/자동화/모임총무/service-worker.js), [`manifest.webmanifest`](/Users/sonjisun/자동화/모임총무/manifest.webmanifest) 기준으로 동작합니다.
- [`app_server.py`](/Users/sonjisun/자동화/모임총무/app_server.py)는 GitHub Pages에서 실행되지 않습니다.

## 1. GitHub에 새 저장소 만들기
- GitHub에서 새 저장소를 만듭니다.
- 저장소 이름은 두 방식 중 하나로 정합니다.
- 일반 저장소: `moim-treasurer`
- 사용자 사이트 저장소: `<github-username>.github.io`

## 2. 이 폴더를 Git 저장소로 연결
```bash
cd /Users/sonjisun/자동화/모임총무
git init
git branch -M main
git add .
git commit -m "Initial treasurer board"
git remote add origin https://github.com/<github-username>/<repo-name>.git
git push -u origin main
```

## 3. GitHub Pages 켜기
1. GitHub 저장소 열기
2. `Settings`
3. 왼쪽 `Pages`
4. `Build and deployment` 에서 `Deploy from a branch`
5. Branch `main`, Folder `/(root)` 선택
6. 저장

## 4. 배포 주소 확인
- 일반 저장소면 주소는 보통 아래 형태입니다.
- `https://<github-username>.github.io/<repo-name>/`
- 사용자 사이트 저장소면 주소는 보통 아래 형태입니다.
- `https://<github-username>.github.io/`

## 5. 아이폰에 설치
1. 아이폰 Safari에서 배포 주소 열기
2. `공유`
3. `홈 화면에 추가`
4. 설치된 아이콘으로 실행

## 6. 데이터 이전
- 기존 기기: `설정 > 관리 도구 > 전체 데이터 백업`
- 새 기기: `설정 > 관리 도구 > 백업 불러오기`

## 주의
- GitHub Pages 버전은 각 기기별 로컬 저장입니다.
- 즉, 아이폰과 PC 데이터는 자동으로 같아지지 않습니다.
- 같은 데이터가 필요하면 백업 파일로 옮겨야 합니다.
