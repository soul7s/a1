# 모임 총무 관리 시스템 기획서

## 1. 문서 개요
- 문서명: 모임 총무 관리 시스템 기획서
- 작성일: 2026-03-08
- 목적: 회비 수납, 지출 기록, 잔액 관리, 월 마감 업무를 총무 1명 기준으로 빠르고 정확하게 처리하는 시스템의 방향을 정의한다.
- 가정: 10명~200명 규모의 동호회, 스터디, 친목 모임 등에서 총무가 실무를 전담한다.

## 2. 배경과 문제 정의
현재 총무 업무는 은행 앱, 메신저, 메모, 스프레드시트에 흩어져 있는 경우가 많다. 이 방식은 총무 입장에서 아래 문제를 만든다.

- 누가 회비를 냈는지 빠르게 확인하기 어렵다.
- 입금자명과 회원명이 달라 매번 수기 대조가 필요하다.
- 지출 내역과 영수증이 따로 관리되어 누락이 생긴다.
- 기초잔액, 현재 잔액, 다음 달 이월금을 한 번에 파악하기 어렵다.
- 월 마감 후 숫자가 바뀌면 어떤 수정이 있었는지 추적하기 어렵다.
- 총무 교체 시 인수인계 자료를 정리하는 데 시간이 많이 든다.

따라서 총무가 실제로 하는 반복 업무를 한 곳에서 처리하고, 기록 누락과 정산 오류를 줄이며, 월 마감과 인수인계까지 안정적으로 관리할 수 있는 장치가 필요하다.

## 3. 목표
### 핵심 목표
- 회비 입금 현황을 빠르게 확인한다.
- 지출 내역과 증빙 자료를 누락 없이 기록한다.
- 기초잔액, 현재 잔액, 기말잔액, 이월금을 자동 계산한다.
- 월 마감 시점의 회계 상태를 재현 가능하게 저장한다.

### 기대 효과
- 총무의 수기 정리 시간을 줄인다.
- 입금 누락, 지출 누락, 잔액 계산 오류를 낮춘다.
- 월별 마감과 인수인계 준비를 빠르게 끝낼 수 있다.
- 회계 수정 이력을 추적할 수 있어 운영 리스크가 줄어든다.

## 4. 대상 사용자
### 1차 사용자
- 총무: 회원 관리, 회비 생성, 입금 확인, 지출 기록, 월 마감, 변경 이력 확인 담당

### 기본 범위 제외
- 대외 공개용 조회 기능
- 외부 공유 링크 기능
- 다중 승인 워크플로우

## 5. 제품 형태 제안
초기 버전은 **모바일에서도 사용 가능한 웹 기반 관리 시스템**으로 설계한다.

선정 이유:
- 설치 없이 바로 접근 가능하다.
- 은행 앱과 번갈아 보며 빠르게 입력하기 쉽다.
- 지출 직후 영수증 사진 첨부가 쉽다.
- 이후 계좌 불러오기, OCR, 자동화 기능으로 확장하기 좋다.

## 6. 핵심 기능 범위
### 6.1 회원 관리
- 회원 등록, 수정, 비활성화
- 이름, 연락처, 닉네임, 입금자명 메모 관리
- 회비 대상 여부 설정

### 6.2 회비 관리
- 월회비, 분기회비, 행사비 생성
- 회원별 납부 상태 관리
- 미납자, 부분 납부자, 확인 필요 상태 집계
- 마감일 관리

### 6.3 입금 처리
- 수동 입금 등록
- 입금자명 기록
- 중복 입금 경고
- 부분 납부, 면제, 확인 필요 처리

### 6.4 지출 관리
- 지출 내역 등록
- 사용일, 금액, 사용처, 카테고리, 메모 입력
- 영수증 이미지 또는 PDF 첨부
- 행사/목적 단위 분류

### 6.5 잔액 및 정산
- 기초잔액 설정
- 총 수입, 총 지출, 현재 잔액 자동 계산
- 월별 수입/지출 요약
- 월 마감 스냅샷 저장
- 다음 달 이월금 계산

### 6.6 운영 보조 기능
- 변경 이력 기록
- 마감 후 수정 제한 또는 재오픈 정책 적용
- 내부 보관용 정산 내보내기

## 7. MVP 범위
초기 출시에서는 아래 기능에 집중한다.

- 회원 관리
- 회비 항목 생성 및 납부 상태 관리
- 입금 수동 등록 및 예외 상태 처리
- 지출 기록 및 영수증 첨부
- 기초잔액 설정
- 대시보드 잔액 표시
- 월별 정산 및 마감 화면
- 변경 이력 기본 로그

MVP에서 제외:
- 실시간 은행 계좌 자동 연동
- 자동 입금자 매칭 고도화
- 대외 공개용 조회 기능
- 외부 공유 링크 기능
- 복수 모임 통합 운영
- 세금계산서/법인 회계 기능

## 8. 주요 사용자 흐름
### 흐름 1. 회비 등록 및 입금 확인
1. 총무가 이번 달 회비 항목을 생성한다.
2. 시스템이 회비 대상 회원에게 납부 상태를 부여한다.
3. 총무가 은행 입금 내역을 보고 입금자를 확인한다.
4. 해당 회원을 납부 완료, 부분 납부, 확인 필요 중 하나로 처리한다.
5. 대시보드의 수입 합계, 미납 인원, 현재 잔액이 즉시 갱신된다.

### 흐름 2. 지출 등록
1. 총무가 지출 발생 직후 내역을 등록한다.
2. 금액, 사용처, 카테고리, 일자, 메모를 입력한다.
3. 영수증 사진 또는 PDF를 첨부한다.
4. 시스템이 현재 잔액에 즉시 반영한다.
5. 정산 화면과 변경 이력에 함께 반영된다.

### 흐름 3. 월 마감 및 이월
1. 총무가 정산 기간을 선택한다.
2. 시스템이 기초잔액, 수입, 지출, 기말잔액을 계산한다.
3. 총무가 월 마감 스냅샷을 저장한다.
4. 다음 기간의 기초잔액은 직전 마감 잔액을 기본값으로 사용한다.

## 11. 운영 정책 초안
- 모든 지출은 증빙 첨부를 원칙으로 한다.
- 회비 납부 여부와 금액 상태는 총무만 변경할 수 있다.
- 수정 및 삭제 이력은 반드시 로그로 남긴다.
- 월 마감 후에는 기존 거래를 직접 수정하지 않고 재오픈 또는 보정 기록 방식으로 처리한다.
- 기초잔액 변경 시 변경 사유를 남긴다.

## 12. 성공 지표
- 월간 회비 정리 시간 50% 이상 감소
- 미납자 확인 시간 70% 이상 감소
- 월별 마감 정리 시간 10분 이내
- 지출 누락 건수 0건에 근접
- 수정 이력 추적 시간 5분 이내

## 15. 결론
이 시스템의 핵심 가치는 **정확성, 속도, 재현 가능성**이다.  
초기에는 총무가 실제로 반복하는 입금 확인, 지출 기록, 잔액 계산, 월 마감 업무를 빠르고 안정적으로 처리하는 데 집중해야 한다.  
특히 기초잔액, 이월금, 수정 이력, 마감 스냅샷은 총무 실무에서 빠지면 안 되는 요소이므로 MVP부터 반영하는 것이 맞다.
